324. Wiggle Sort II

题目大意

给定一个无序的数组nums，要求你对它进行重新排序，使其满足以下要求：

解题思路

从题目给定的意思，重排序的数组，数组奇数位(位置1, 3, 5,…)的数要大于左右两边的数。要满足这个条件，我们可以对nums进行从小到大排序。然后把数组分成两部分，前面的部分数一定会小于后面部分的数。既对于长度为偶数的数组，假设长度为2n, 那么前面的部分为0 ~ n - 1, 后面部分为n ~ 2n - 1。对于长度为奇数的数组，假设长度为2n + 1, 那么前面的部分为0 ~ n, 后面的部分为n + 1 ~ 2n。然后交替的从前部分取一个数，再从后面部分取一个数，重新组成一个数组，既可以形成题目要求的排序结果。这种方法时间消耗在排序上，平均耗时为O(nlogn)，空间消耗为O(n)。
我们还可以用另外一种方法来时平均耗时降到O(n), 空间消耗降到O(1)。我们不需要对整个数组进行从小到大的排序，只要把数组可以分成两部分，保证前面部分都所有数都小于后半部分即可。所以我们可以求出第k大的数，如果数组里面的数大于这个数，则属于后半部分。若小于等于，则属于前半部分。既当数组长度为2n时，k为n。当数组长度为2n + 1时，k为n + 1。我们可以用快排的思想，找到第k大的数。平均时间消耗为O(logn)。然后我们可以用坐标映射的方式来完成数的交换。用指针p1表示前半部分，随着它的位置增加，可以映射到数组的偶数位上。指针p2表示后半部分，随着它的位置增加，可以映射到数组的奇数位上。所以顺序扫描数组，如果当前数大于之前求出的第k的数，则将这个数交换到p2映射的位置，p2增加。反正，交换到p1映射到的位置。此时时间消耗为O(n),空间消耗为O(1)。
